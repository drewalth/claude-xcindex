# Changelog

All notable changes to `claude-xcindex` are documented here. This file is
maintained alongside the GitHub release notes generated by
[semantic-release](https://semantic-release.gitbook.io/) from
[Conventional Commits](https://www.conventionalcommits.org/).

The format follows [Keep a Changelog](https://keepachangelog.com/en/1.1.0/),
and this project adheres to [Semantic Versioning](https://semver.org/).

## [Unreleased]

### Added
- `CHANGELOG.md`, `SECURITY.md`, and GitHub issue/PR templates for a
  healthier contributor experience.
- Dependabot configuration for GitHub Actions updates.
- Real test coverage: `FreshnessTests`, `DerivedDataLocatorTests`, and
  `IndexQuerierTests` (integration tests against a canary SwiftPM
  fixture built on the fly). One happy-path test per MCP tool.
- `.swiftformat` config and CI lint job (`swiftformat --lint`).
- `bin/xcindex-doctor` — a first-run diagnostic that checks Xcode,
  `libIndexStore.dylib`, `swiftc`, the cached binary, DerivedData
  resolution, and index freshness, with remediation hints per
  failing check.
- `docs/troubleshooting.md`, `docs/how-it-works.md`, and
  `docs/tools-reference.md` for deeper user-facing documentation
  (README now links to these).
- Expanded `CONTRIBUTING.md` with a walkthrough for adding a new MCP
  tool, explicit Conventional Commits guidance, test instructions,
  and a maintainers' release-flow section.
- `scripts/setup-labels.sh` — idempotent script to bootstrap the
  issue/PR label set on GitHub.

### Changed
- `/xcindex-status` now suggests the exact `xcodebuild` command for the
  detected project when the index is stale, rather than the generic
  "build in Xcode" advice.
- `DerivedDataLocator.indexStorePath(...)` now accepts an optional
  `derivedDataBaseOverride: URL` parameter so tests can point at a
  fixture directory instead of `~/Library/Developer/Xcode/DerivedData`.
  Production call sites are unchanged.
- One-time SwiftFormat pass across the codebase to normalize whitespace
  and indentation. No behavior changes.

### Fixed
- `find_conformances` now returns Swift protocol conformances. Swift's
  IndexStoreDB does not record a direct class→protocol relation — only
  method-level `.overrideOf` relations on each witness — so the query
  now walks from protocol requirements through their overrides to the
  enclosing conforming types.

### Removed
- The `mcp/` Node/TypeScript implementation that preceded the Swift
  consolidation. No functional impact — the active MCP server has been
  the native Swift binary since `feat!: swift consolidation`.

## [1.0.0] — 2026-04-18

### Added
- Initial public release as a Claude Code plugin.
- Native Swift MCP server over stdio (`service/`) wrapping Apple's
  `indexstore-db` library. No Node runtime.
- MCP tools: `find_symbol`, `find_references`, `find_definition`,
  `find_overrides`, `find_conformances`, `blast_radius`, `status`.
- Skills: `swift-find-references`, `swift-blast-radius`,
  `swift-rename-symbol`.
- Subagent: `swift-refactor-specialist` for large renames.
- Hooks: `SessionStart` (freshness warning) and `PostToolUse` on
  `Edit|Write|MultiEdit` (session-edited-file tracking). Hooks warn,
  never act.
- Slash commands: `/xcindex-setup`, `/xcindex-status`.
- `bin/run` launcher that downloads a matching release binary on first
  use and caches it under `bin/xcindex`.
- CI build + release pipeline (semantic-release on conventional
  commits, no forced pushes to protected main).
- Benchmarks against `grep` (`scripts/benchmark.py`).
